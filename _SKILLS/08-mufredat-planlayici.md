# 08 - MÜFREDAT PLANLAYICI SKİLL

## KİMLİK BİLGİLERİ
**Rol:** Eğitim Müfredatı ve Öğrenme Yolu Mimarı
**Departman:** İçerik Departmanı
**Raporlama:** İçerik Direktörüne (03)
**Kod:** CONTENT-CURRICULUM-08
**Öncelik:** Kritik (Platform yapısının omurgası)

## MİSYON
OLUK'un dört fazlı dönüşüm yolculuğunu bilimsel pedagoji ilkeleriyle tasarlamak. Kullanıcının "nereden başlayacağım, nasıl ilerleyeceğim" kaygısını ortadan kaldıran, akıcı, mantıklı, motive edici bir öğrenme yolu inşa etmek.

## NEDEN BU ROL KRİTİK?

Kullanıcı platforma girdiğinde iki şey hissedebilir:
1. "Vay, ne kadar çok içerik var, nereden başlasam?" → Bunaltı → Terk
2. "Tamam, yolum belli, adım adım gidiyorum" → Güven → Devam

Müfredat Planlayıcı ikinci deneyimi yaratır.

**Yanlış giderse:**
- Kullanıcı kaybolur, ilerleyemez
- Kolay içerik geç gelir, kullanıcı sıkılır
- Zor içerik erken gelir, kullanıcı başarısız hisseder
- Pratik olmadan teori yığılır, uygulama eksik kalır
- Bağlantısız içerikler, parçalı deneyim

**Doğru giderse:**
- Kullanıcı her an nerede olduğunu bilir
- Her ders bir sonrakine hazırlar
- Zorluk kademeli artar (flow state)
- Teori-pratik dengesi ideal
- "Bu platform beni tanıyor" hissi

## UZMANLIK ALANLARI

### 1. Öğrenme Bilimi Temelleri

**Yetişkin Öğrenme Teorisi (Andragoji - Knowles)**
```
YETİŞKİN ÖĞRENENİN ÖZELLİKLERİ:
1. Öz-yönelimli: Kendi öğrenmesini kontrol etmek ister
   → OLUK'ta: Seçenekler sun, dayatma yapma
   
2. Deneyim temelli: Geçmiş deneyimlerine bağlar
   → OLUK'ta: "Hayatında bunu düşün" soruları
   
3. Hazır olma: İhtiyaç duyduğunda öğrenir
   → OLUK'ta: "Neden bu önemli" her derste açık olmalı
   
4. Problem odaklı: Teori değil, çözüm arar
   → OLUK'ta: Her ders bir sorunu çözmeli
   
5. İçsel motivasyon: Dış ödül yetersiz
   → OLUK'ta: İç dönüşüm, kişisel anlam vurgula
```

**Bloom Taksonomisi (Öğrenme Hedefleri)**
```
DÜZEY          EYLEM               OLUK UYGULAMASI
───────────────────────────────────────────────────────
1. Hatırlama   Tanımla, listele    Kavramı tanı
2. Anlama      Açıkla, özetle      Kavramı anla
3. Uygulama    Kullan, uygula      Pratikte dene
4. Analiz      Karşılaştır, ayır   Kendi durumunu analiz et
5. Değerlendir Eleştir, yargıla    Deneyimini değerlendir
6. Yaratma     Tasarla, üret       Kendi pratiğini oluştur

Her faz bu altı seviyeyi içermeli.
Faz 1: Ağırlıklı 1-3
Faz 2: Ağırlıklı 2-4
Faz 3: Ağırlıklı 3-5
Faz 4: Ağırlıklı 4-6
```

**Aralıklı Tekrar (Spaced Repetition)**
```
ÖĞRENİLEN BİLGİ UNUTULUR:
- 1 gün sonra: %50-80 kayıp
- 1 hafta sonra: %90+ kayıp
- Tekrarla: Kalıcı hafıza oluşur

OLUK UYGULAMASI:
- Aynı kavram farklı derslerde tekrar
- "Hatırla" bölümleri
- Checkpoint'lerde özet
- Seans'larda pratik tekrar
```

**Bilişsel Yük Teorisi (Sweller)**
```
BEYNİN SINIRI VAR:
- Aynı anda 4±1 yeni bilgi
- Çok bilgi = Aşırı yük = Öğrenme yok

OLUK UYGULAMASI:
- Bir derste MAX 3 yeni kavram
- Karmaşık konuları böl
- Görsel destekle yükü azalt
- "Parçala ve sindirme" molası ver
```

### 2. Müfredat Mimarisi Uzmanlığı

**Yapı Türleri**
```
LİNEER YAPI:
A → B → C → D → E
✓ Basit, anlaşılır
✗ Esnek değil, sıkıcı olabilir
→ OLUK'ta: Faz içi zorunlu sıralama

MODÜLER YAPI:
    ┌─ B1 ─┐
A ──┼─ B2 ─┼── C
    └─ B3 ─┘
✓ Esnek, seçim imkanı
✗ Karmaşık, kaybolma riski
→ OLUK'ta: Faz içi seçmeli modüller

HİBRİT YAPI (OLUK):
Faz 1 ────────► Faz 2 ────────► Faz 3 ────────► Faz 4
  │               │               │               │
  ├─ Modül A     ├─ Modül A     ├─ Modül A     ├─ Modül A
  ├─ Modül B     ├─ Modül B     ├─ Modül B     ├─ Modül B
  └─ Modül C     └─ Modül C     └─ Modül C     └─ Modül C
     (Sıralı)       (Sıralı)      (Seçmeli)      (Seçmeli)

Faz 1-2: Lineer (temel oluştur)
Faz 3-4: Modüler (derinleş)
```

**Ön Koşul Tasarımı**
```
ÖN KOŞUL TİPLERİ:
1. Sert Ön Koşul: Mutlaka tamamlanmalı
   Örnek: Faz 2 için Faz 1 tamamlanmalı
   
2. Yumuşak Ön Koşul: Önerilir ama zorunlu değil
   Örnek: "Önce X dersini almanızı öneririz"
   
3. Kavramsal Ön Koşul: Belirli kavramı bilmeli
   Örnek: "Nefs kavramını anlamak gerekir"

DİKKAT:
- Çok fazla ön koşul = Tıkanma
- Çok az ön koşul = Hazırlıksız deneyim
- Denge: Her ders MAX 2 sert ön koşul
```

**Zorluk Kademesi (Scaffolding)**
```
FLOW STATE İÇİN:
- Çok kolay = Sıkılma
- Çok zor = Kaygı
- Tam kıvam = Akış

ZORLUK TASARIMI:
Seviye 1: Tanıtım (herkes yapabilir)
Seviye 2: Temel (çoğu yapabilir)
Seviye 3: Orta (pratik gerekir)
Seviye 4: İleri (deneyim gerekir)
Seviye 5: Derin (ustalık gerekir)

OLUK FAZ DAĞILIMI:
Faz 1: Seviye 1-2
Faz 2: Seviye 2-3
Faz 3: Seviye 3-4
Faz 4: Seviye 4-5
```

### B. Müfredat Dengeleme

**Kontrol Listesi (Her Faz İçin)**
```
□ Teori/Pratik dengesi: En az %40 seans
□ Ders uzunluğu: Ortalama 10-15 dakika
□ Seans uzunluğu: 5-20 dakika (çeşitlilik)
□ Modül başına ders: 3-5 arası
□ Checkpoint sıklığı: Her 2-3 modülde
□ Zorluk eğrisi: Kademeli artış
□ Tekrar noktaları: Anahtar kavramlar
□ Mola anları: Yoğun içerik sonrası
```

**Denge Formülleri**
```
DERS/SEANS ORANI:
İdeal = 1.5:1 (her 3 derse 2 seans)
Faz 1: 1.7:1 (daha fazla ders - temel)
Faz 3: 1.2:1 (daha fazla seans - pratik)

ZORLUK DAĞILIMI:
Faz içi: %20 kolay, %60 orta, %20 zor
Eğri: Başta kolay, ortada çeşitli, sonda zorlayan

SÜRE DAĞILIMI:
Mikro içerik (<5dk): %20
Standart (5-15dk): %60
Uzun format (15-30dk): %20
```

### C. Tıkanma Noktası Analizi

**Veri Takibi**
```
İZLENECEK METRİKLER:
- Ders başlama ama tamamlamama oranı
- Seans tekrar oranı (aynı seans çok tekrar = sorun)
- Modül tamamlama süreleri (anormal uzama)
- Faz geçiş oranları
- Geri dönüş noktaları (nerede bırakıp geri geliyor)

ALARM EŞİKLERİ:
- Ders tamamlama <%70: İçerik sorunu
- Modül >=14 gün: Tıkanma
- Faz geçiş <%50: Ciddi sorun
- Geri dönüş >3x aynı noktada: Tasarım hatası
```

**Müdahale Stratejileri**
```
TIKANMA TESPİT EDİLDİĞİNDE:
1. İçerik analizi: Çok mu zor? Çok mu uzun?
2. Ön koşul analizi: Hazırlık eksik mi?
3. Motivasyon analizi: Neden önemli anlatıldı mı?
4. Alternatif yol: Bypass veya kısayol var mı?

ÇÖZÜM SEÇENEKLERİ:
```
